Extension Help and Configuration Guide

Quick Start

  1. Pick Sync Mode based on your workflow (Plugin Endpoint or Direct API).
  2. Enter InvenTree Base URL and API Token.
  3. For direct mode, set Default Category ID at minimum.
  4. Capture a page, review preview rows, then Send To InvenTree.
  5. Use Direct Mode Dry-Run Validation before your first sync.

What This Extension Does

  • Captures product rows from supported supplier pages.
  • Exports captured rows to JSON or CSV files.
  • Sends captured data to InvenTree using Plugin Endpoint mode or Direct API mode.
  • Optionally syncs images, supplier-part records, and stock records.

Basic Workflow

  1. Open a supported source page.
  2. Click Capture Current Page.
  3. Review captured rows in the preview area.
  4. Either download JSON/CSV or click Send To InvenTree.

Sync Mode

  • Plugin Endpoint: Sends one payload to your custom endpoint path.
  • Direct InvenTree API: Creates/updates parts directly via native InvenTree API routes.

Core InvenTree Settings

  • InvenTree Base URL: e.g. https://inventree.local or https://inventree.example.com
  • API Token: personal or service token with create/update permissions.
  • Plugin Endpoint Path: used only in Plugin Endpoint mode.

Direct API Settings

  • Part API Path: default /api/part/
  • Supplier Part API Path: default /api/company/part/
  • Stock Item API Path: default /api/stock/
  • Default Category ID: required in direct mode to create new parts.
  • Default Supplier ID: needed when syncing supplier-part records.
  • Default Stock Location ID: needed when creating stock items.
  • Stock Quantity Header Hint: optional source column for quantity (for example Qty).
  • Default Stock Quantity: fallback quantity when no value is found from source data.

Header Mapping Hints

Use these to map uncommon source table headers:

  • Name header
  • Description header
  • MPN header
  • Supplier PN header
  • Image URL header

Images

  • Include image URL in exported/sent items adds image_url to payload and exports.
  • Upload images to InvenTree uploads images when part IDs are available after sync.
  • Part upload path defaults to /api/part/{id}/upload/.

Existing Match Strategy

  • Skip matching existing entries: avoids duplicate updates.
  • Update matching existing entries: updates existing records when exact matches are found.

Example Config: Plugin Mode

Sync Mode: Plugin Endpoint
InvenTree Base URL: https://inventree.local
API Token: <token>
Plugin Endpoint Path: /api/plugin/product-import/
Include image URL: On
Upload images: On

Example Config: Direct API Mode

Sync Mode: Direct InvenTree API
InvenTree Base URL: https://inventree.local
API Token: <token>
Part API Path: /api/part/
Supplier Part API Path: /api/company/part/
Stock Item API Path: /api/stock/
Default Category ID: 12
Default Supplier ID: 4
Default Stock Location ID: 7
Default Stock Quantity: 1
Sync supplier-part records: On
Create stock items: Off

Troubleshooting

  • No rows found: verify page has a visible product table and re-capture.
  • 401/403: token missing scope or expired.
  • 400 on direct mode: check category/supplier/location IDs and API paths.
  • Image upload skipped: ensure image URLs are present and uploads are enabled.
